v0.1 · FOR CLAUDE CODE

Inside the CRT.

Circle Waifu is a Farcaster mini-app: a retro research-lab terminal where an anime companion gives one useful onchain mission per day. This document is the implementable design system — tokens live in tokens/circle-waifu.css, and every screen composes from the .cw-* primitives below. Do not hardcode hex values in route code — reference the tokens.

Visual DNA: full-bleed "inside the screen" immersion (à la sible.ai) × Tamagotchi companion loop × 80s phosphor terminal. Cyan/electric-blue phosphor signal, hot-magenta waifu accent, medium scanlines, pixel labels over crisp mono body.

04

CRT Effect Layers

Wrap any full-bleed container in .cw-crt. It paints scanlines (::before) and a vignette + phosphor wash (::after); children auto-lift above the texture. Medium intensity by default — readable, not nauseating. Add .cw-crt--flicker for a gentle flicker and a .cw-scanbeam child for a sweeping beam (both auto-disable under prefers-reduced-motion).

SYSTEM ONLINE PHOSPHOR · SCANLINES · VIGNETTE · FLICKER awaiting field experiment
Scanline opacity is tokenized (--cw-scanline-opacity: .14 dark / .05 light) so the texture lightens automatically per theme. Never apply CRT FX to dense text blocks like the activity log — keep it on the stage, dock, and waifu viewport.
05

Buttons

All buttons are pixel-labelled, ≥44px tall, with snappy press states. Hierarchy: --primary (magenta, the day's CTA — one per screen), --phosphor (system actions), default (neutral), --ghost (tertiary).

Satellite icon buttons (float around the waifu / dock)

Icons shown as glyph placeholders. Production: swap for a pixel icon set (16×16, 2px stroke) — keep stroke weight equal to the pixel grid so they read as part of the terminal.

Waifu Viewport & Moods

The hero element. A 3:4 CRT-framed slot (.cw-waifu) holds a higher-res anime portrait. The frame's data-mood tints the border & glow so her state is felt before it's read. In the prototype this is an <image-slot> you can drop real art into; here it shows the striped placeholder.

data-mood="hyped"
WAIFU ART 3:4 portrait slot
data-mood="happy"
WAIFU ART drop render
data-mood="neutral"
WAIFU ART drop render
data-mood="sad"
WAIFU ART streak broken
Mood mapping → hyped: just verified a mission / won a draw (magenta). happy: streak active (green). neutral: idle, mission pending (cyan). sad: streak broken / missed day (desaturated). The waifu is user-nameable — store the name and render it in the top-left HUD; default placeholder is [WAIFU].

11

App Shell & Home Anatomy

Full-bleed, single column, capped at --cw-app-max (480px) for the Farcaster mini-app. No headers, no nav rail — "nothing else anywhere." The waifu owns the stage; HUD chips orbit her; the day's mission card sits over the lower third; a blurred dock anchors navigation + the primary CTA.

Never infantilize. Labels are clipped & uppercase (TODAY'S HYPOTHESIS, CHAIN ACTION, STREAK INTEGRITY). Use "weekly prize pool," never casino language. Always show odds, caps and rules plainly.

Motion

Fast & stepped, never bouncy. Transitions --cw-dur 220ms on --cw-ease; meters use --cw-ease-step for a refreshing-display feel. Decorative loops (flicker, scanbeam, cursor blink) only — gate everything on prefers-reduced-motion. No infinite animation on content.

Implementation contract for Claude Code

1 — Import tokens/circle-waifu.css once at the app root; set data-theme on <html>.

2 — Build screens only from .cw-* primitives + tokens. New component? Add it to the token file / DS registry first, then compose.

3 — Never hardcode color/space/type values in route code. Reference var(--cw-*).

4 — Waifu portrait is a swappable slot; cosmetics are token overrides, not new CSS.

5 — Honor prefers-reduced-motion and env(safe-area-inset-*) everywhere.
